# 🚀 Launching FlowEstate AI

This guide covers how to deploy the FlowEstate AI application for public viewing.

## 1. Deploy to Vercel (Recommended)
Next.js works best on Vercel. 

1. **GitHub Setup:** Push your code from this folder to a new GitHub repository.
2. **Sign in to Vercel:** Go to [Vercel.com](https://vercel.com).
3. **Import Project:** Click **"New Project"** and import your GitHub repository.
4. **Choose a Unique Name:** 
   > [!IMPORTANT]
   > Do NOT just type "flow-estate" as that name is already taken by a different project (an NFT site you saw). 
   > Choose a unique name like `blakes-flow-estate` or `flow-estate-2026`. This will be your sub-domain (e.g., `your-name.vercel.app`).
   
5. **Environment Variables:** You MUST add these in the Vercel dashboard under **Settings > Environment Variables**:
   - `GEMINI_API_KEY`
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`

6. **Deploy:** Click **Deploy**. Vercel will give you a unique link that is yours!

## 2. Production Build Check
Before deploying, it is best practice to run a production build locally to ensure no errors:

```bash
npm run build
```

Then you can start the production server:

```bash
npm run start
```
